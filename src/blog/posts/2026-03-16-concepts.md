## Things that Confuddle My Poor Mind
### i++ V.S. ++i
i++ performs the increment after the expression runs. ++i increments the variable, then performs the expression.

Let's take `for (int i = 0; i < array.size(); i++)` and `for (int i = 0; i < array.size(); ++i)`
If we jot down the values of each variable in each pass:

1. `for (int i = 0; i < array.size(); i++)` starts off with 0, finishes the expression inside the loop, then increments it upwards. 0, 1, 2, ... n.
2. `for (int i = 0; i < array.size(); ++i)` starts off with 1, as the increment happens first. 0 + 1 is 1, then 1 is passed into the expression. 1, 2, 3, ... n.

### Lambda Functions
Now lambda functions are actually something I'm more unfamiliar with, so let's break down the syntax.

`[](int a) -> int { return a - 1; };`
Where `[]` defines the start of the lambda function. But we also want to know what variables the lambda function can access. `[]` captures nothing, `[x]` captures x by value, `[&x]` captures x by reference, `[=]` captures all by value, `[&]` captures all by reference.

`(int a)` is the parameter list, `int {...}` is the return type, and the inside is obviously the function body.

Lambda functions are one way you are able to define functions within another function -- prior to C++11, all functions were global.

### Ternary Operator
These were actually pretty intimdating to come across, but once you realise it's just another format of an if-expression, it gets much less scary.

Let's take `int maxNumber = (a > b) ? a : b;` as an example, and break it down slowly.

`(a > b)` is the expression we want to evaluate. I like to think of the question mark as asking a literal question. For instance, is `(a > b)` true? If yes, then `a` is "returned", otherwise, `b` is returned. That's it!

I feel like one of my only problems is that I don't actually think of using them often, causing me to forget what they do or avoid them in general. I guess I'll try implementing them into my Leetcode questions more often!

### Using Pass-By-Reference
I've studied this concept through my flashcards, and I do know it, very theoretically, in words. I know what it does -- you're storing the reference instead of the actual object, into a variable. That way, you store the memory address instead of the object, saving space, and all that sweet stuff. I also know the syntax, which is, for example, `int& a = x`. In this case, a is the new variable we are creating, connecting it to the *memory address* of x through the ampersand (&) symbol.

However, I have not really created functions from scratch with the pass-by-reference in mind.
